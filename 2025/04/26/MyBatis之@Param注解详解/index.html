1.认识@Param注解

@Param注解通常用于 MyBatis（一个流行的 Java 持久层框架）中，用于给 SQL 映射方法中的参数起别名，便于在 XML 映射文件中引用。这是 MyBatis 中的一种常见用法，尤其在方法参数较多时，使用 @Param 可以明确标识参数的名称。

3.为什么要使用@Param注解？

1.命名清晰：当方法有多个参数时，使用 @Param 为每个参数命名可以清晰地知道每个参数的含义，避免参数混乱。

2.避免位置依赖：如果不使用 @Param，MyBatis 会根据参数的位置来推断它们的名称。但如果方法参数顺序发生变化，SQL 映射可能会出错。通过 @Param 可以避免这种问题，因为每个参数都显式地指定了名称。

3.提高可读性和维护性：在 SQL 映射文件中，可以通过清晰的参数名称提高代码的可读性，使得后续的维护更加方便。

5.总结

@Param 注解在 MyBatis 中的作用是帮助映射方法的多个参数，使它们能够在 SQL 语句中有明确的名字。对于没有 @Param 的方法参数，MyBatis 会将它们打包成一个 Map 来传递给 SQL 语句。使用 @Param 注解能够确保每个参数都有唯一的标识，并避免多参数情况下的冲突问题。

在程序调用时执行到目标方法后，底层会建立一个SortedMap<Integer, String>类型的集合，称names，其中key是序号(从0开始),value是@Param中指定的参数名;然后遍历names，将names中所有的参数名作为key，传递的参数值作为value存储到一个Map<String, Object> 中，称param，最后将param返回。
